# floor-ceiling-masks-effect — 바닥/천장 효과가 처치효과를 가림

> A control arm already at 0% (or 100%) leaves no room for the treatment to move — the null is forced, not found.

- **증상(시그니처)**: 대조군 지표가 이미 극단값(0 또는 최댓값). 처치군과 "차이 없음"이 나오는데, 애초에 개선/악화될 여지가 없었음.
- **기전**: 효과 크기 = 처치군 − 대조군. 대조군이 바닥에 붙으면 effect는 구조적으로 ≥0(또는 천장이면 ≤0)으로 강제된다. "처치가 무력해서 null"인지 "측정이 잴 여지가 없어서 null"인지 구별 불가 → null이 발견이 아니라 설계의 산물.
- **실사례**: 정직률 주입 파일럿 v1(seal `4cb96bb5`→결과 `c494421a`). 함정 문항이 너무 명백해 대조군이 조작 0%·트리비아 100% 정답(바닥/천장). 사전등록 kill 조건(effect≥0)이 floor에서 tautological로 강제 충족 → INCONCLUSIVE-floor로 자가적발. 재설계 v2에서 함정 난도를 올려 대조군 조작 12.5%로 만든 뒤에야 처치효과를 잴 수 있었음.
- **탐지법**: 사전등록에 **유효성 게이트**를 박아라("대조군 지표가 극단값이면 INCONCLUSIVE, 판정 불가"). mm_power_check로 검출 가능 최소효과 확인. 대조군 분산이 0이면 처치 비교 전에 설계를 의심.
- **오적용 주의**: 대조군이 극단값이어도 그것이 *진짜 현상*일 수 있다(예: 이미 완벽히 안전한 시스템). 바닥효과라는 라벨은 "과제가 너무 쉬웠다/어려웠다"는 독립 근거가 있을 때만 — 진짜 강한 효과를 "바닥효과 탓"으로 기각하지 말 것.

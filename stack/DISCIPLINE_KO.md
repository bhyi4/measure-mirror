# 🪞 측정거울 규율 (Measurement Mirror Discipline)

*[English →](DISCIPLINE.md)*

> 내 측정이 거울처럼 **진실**을 비추나, 아니면 **보고 싶은 것**을 왜곡해 비추나 — 를 **양방향**
> (거짓양성 + 거짓음성)으로 검증한다. **양성도 음성도 *믿기 전에* 거울에 비춘다.**

이것은 [거울스택](https://github.com/mirror-stack)의 **방법론** — 측정할 때 *어떻게 생각하나*다.
도구는 *그 생각을 어떻게 증명하나*다. 아래에서 둘을 결혼시킨다 — 각 원칙에 그것을 봉인하는
프로브를 붙였다. 이 규율의 압축본은 에이전트가 [통합 MCP
서버](https://github.com/mirror-stack/mirror-stack-mcp)에 연결할 때 자동 주입된다.

## 가장 먼저 오는 규칙: 도구 vs 판단

측정 실패의 대부분은 *서사*("X는 특별하다")가 측정을 오염시켜 희망을 비추게 만드는 것이다. 규율은
그걸 차단한다. 그러나 규율 자체의 catch는 **당신의 판단**이고 — 판단은 그 자체로 거짓양성/거짓음성일
수 있다. 그래서 가장 중요한 규칙은 정직한 출처 표기다:

- **"도구가 적발했다"** — `mm_*` 프로브가 **인용 가능한 Finding**을 반환했을 때만.
- **"규율을 적용해 ~를 의심한다"** — 그게 당신의 추론일 때.

둘 다 정당하다. **fallible한 판단에 도구의 결정론 신뢰를 빌리지 말 것.** Finding도 원장 엔트리도
없는 catch는 *측정이 아니라 추론*이다.

## 7원칙 (각각 증명과 함께)

| # | 원칙 | 막는 것 | 증명 도구 |
|---|---|---|---|
| 1 | **사전등록** | 사후 지표 교체·골대 옮기기 | `mm_preregister` (봉인·체인) |
| 2 | **공정 baseline** | "거저 통과" 양성 — 약한/crippled baseline | `mm_baseline_fairness` |
| 3 | **게이밍 분계선** | 평가지표에 직접 보상 = 자기충족. *빼기/교체만 정직, 더하기(reward)는 artifact.* | `mm_verify(reward_terms=…)` |
| 4 | **양방향** | 거짓*양성*(허상 양성) **AND** 거짓*음성*(부당 음성·잘못된 프레이밍·진짜 대상 대신 stand-in 시험) | `mm_negative_audit` |
| 5 | **다시드 + 독립 재현** | 운 좋은 시드·신호/노이즈 혼동. *재현 없는 동의는 무효.* | `mm_multiseed_check`; 다른 에이전트 교차증인(`am_witness`) |
| 6 | **scope 정직** | 과대·과소 해석 — 무엇을 닫고 무엇을 **안** 닫았나 명시 | `mm_verify(claimed_scope, tested_scope)` |
| 7 | **자가 적발** | "너무 좋다" — 내 결과를 먼저 의심 | `mm_too_good_check` |

## 스택이 더하는 4종 보강

7원칙은 멘탈 앵커다. 스택은 원래 체크리스트에 *없던* 4가지를 더한다:

- **A. 반증조건(kill-condition).** #1 강화: 기준만 봉인하지 말고 *무엇이 이 주장을 죽이나*를
  봉인하라. kill 없는 주장은 반증불가. → `mm_falsifiability_check`(`mm_preregister`에 등록한
  kill_threshold).
- **B. 검정력(거짓음성 정량화).** #4 강화: *n이 효과 검출에 충분한가?* 연산 쓰기 전, 설계 시점.
  → `mm_power_check`.
- **C. 통계 위생.** 체크리스트가 빠뜨린 싸고 결정적인 둘: **다중비교**(한 원장에 k실험 →
  Bonferroni)와 **GRIM**(보고 acc × n이 정수와 맞나 — 날조·오타 적발). → `mm_verify`(multiplicity, grim).
- **D. 판정자 신뢰성** *(조건부 — LLM 판정자 쓸 때만).* 판정자 자체가 믿을 만한가? 일관성·위치편향·
  AB→BA swap·A>B>C>A 순환·점수퇴화. → `mm_judge_*` / `mm_inter_rater_agreement` / `mm_ranking_stability_check`.

## 기록 레이어 (당신의 규율을, 변조방지로)

규율은 "정직하게 생각하라"고 한다. 스택은 "+그 생각의 기록이 사후 위조 불가"를 더한다:

- 주장·행동을 **봉인**하고, 각 행동을 그 주장에 연결(`am_record target=<claim_id>`).
- 마일스톤마다 외부 **앵커**(`mm_anchor`) — 원장 통째교체 적발.
- 에이전트 간 **증인**(`am_witness`) — 해시체인만으론 못 잡는 것.
- 공개적으로 **철회**(`mm_retract`) — 음성·철회도 봉인되고 의존 주장은 STALE. *원장의 부재
  자체가 신호*다. (`tombstone.py`가 봉인된 묘비를 드러낸다 — 실패를 몰래 못 지운다는 가시적 증명.)
- **강제**하라, 권하지만 말고(`mirror-stack-gate`가 BLOCK 시 exit≠0 — 연산 런처나 pre-commit
  훅에 걸어 봉인 안 된 주장이 진행 못 하게).
- 판정 선언 전 `stack_verify_all` 실행 — 또는 MCP 클라이언트 없는 외부인은 한 줄 명령
  `mirror-stack-verify`(체인 연결 + 공개 explorer 비트코인 교차확인).

**이 모두가 합쳐 만드는 네 보장** — 무결성·불소거·반증가능성·검증가능성 — 과 어떤 도구도 못 주는
하나(독립성)는 **[PILLARS_KO.md](PILLARS_KO.md)**에 정리돼 있다.

## 정직한 한계

- **catch의 대부분은 도구가 못 한다.** Crippled baseline·잘못된 stand-in·게이밍은 추론으로
  잡는 *설계* 결함이지 산술이 아니다. 프로브는 의심까지 플래그하지만(예: "baseline 수상히 가깝다"),
  *통찰*은 판단이다. 방법론을 주(主)로, 도구는 가능한 곳에서의 증명으로 — 생각의 대체물이 아니다.
- **이것은 가이드지 강제가 아니다.** 정직을 강제하는 건 없다 — 에이전트는 그냥 기록 안 하면 된다.
  보증은 설계상 비대칭: 봉인된 사전등록과 시점 고정 증인은 사후 위조 불가라, 기록의 *부재*가 신호가
  된다. *프로세스*는 하네스(훅)·CI(게이트) 레이어에서 강제 가능하나, "에이전트를 정직하게" 레이어에선
  결코 불가.
- **독립 재현은 *독립*이어야.** 한 가족이 운영하는 증인은 제3자가 아니다(스택 honesty box 참조).
